NativeLink provides full support for Buck2's Remote Execution API, enabling distributed builds and caching for Buck2 projects.

## Prerequisites

- Buck2 installed (latest stable version)
- A running NativeLink instance (see [Quickstart](/quickstart))
- Basic understanding of Buck2 build configuration

## Configuration

Buck2's remote execution is configured through the `.buckconfig` file in your project root.

### Basic Setup

<Steps>
  <Step title="Create or update .buckconfig">
    Add the following remote execution configuration:

    ```ini .buckconfig
    [cells]
    root = .

    [buck2_re_client]
    engine_address = localhost:50051
    action_cache_address = localhost:50051
    cas_address = localhost:50051
    tls = false
    instance_name = main

    [build]
    execution_platforms = root//platforms:platforms
    ```
  </Step>

  <Step title="Create platform definitions">
    Define your execution platforms in `platforms/BUCK`:

    ```python platforms/BUCK
    load(":defs.bzl", "platforms")

    platforms(
        name = "platforms",
    )
    ```

    Create `platforms/defs.bzl`:

    ```python platforms/defs.bzl
    def platforms(name):
        native.platform(
            name = "linux-x86_64",
            constraint_values = [
                "ovr_config//os/constraints:linux",
                "ovr_config//cpu/constraints:x86_64",
            ],
        )

        native.platform(
            name = "platforms",
            deps = [":linux-x86_64"],
        )
    ```
  </Step>

  <Step title="Create .buckroot">
    Ensure your project has a `.buckroot` file in the repository root (can be empty):

    ```bash
    touch .buckroot
    ```
  </Step>
</Steps>

## Running Buck2 with NativeLink

<Steps>
  <Step title="Start NativeLink">
    ```bash
    nativelink buck2_cas.json5
    ```
  </Step>

  <Step title="Build with Buck2">
    Buck2 automatically uses remote execution when configured:

    ```bash
    buck2 build //...
    ```
  </Step>

  <Step title="Verify remote execution">
    Check Buck2's output for remote execution indicators:

    ```
    Action stats:
    - Remote executions: 42
    - Cache hits: 15
    ```
  </Step>
</Steps>

## Configuration Options

### Remote Execution Client

The `[buck2_re_client]` section supports the following options:

<CodeGroup>
```ini Basic
[buck2_re_client]
engine_address = localhost:50051
action_cache_address = localhost:50051
cas_address = localhost:50051
tls = false
instance_name = main
```

```ini With TLS
[buck2_re_client]
engine_address = cache.example.com:50051
action_cache_address = cache.example.com:50051
cas_address = cache.example.com:50051
tls = true
tls_ca_cert = /path/to/ca.crt
tls_client_cert = /path/to/client.crt
tls_client_key = /path/to/client.key
instance_name = main
```

```ini Separate Services
[buck2_re_client]
# Separate endpoints for different services
engine_address = scheduler.example.com:50052
action_cache_address = cache.example.com:50051
cas_address = cas.example.com:50051
tls = false
instance_name = production
```
</CodeGroup>

### Key Configuration Fields

| Field | Description |
|-------|-------------|
| `engine_address` | Remote execution scheduler endpoint |
| `action_cache_address` | Action cache service endpoint |
| `cas_address` | Content Addressable Storage endpoint |
| `tls` | Enable TLS (true/false) |
| `instance_name` | Instance name for multi-tenant setups |
| `tls_ca_cert` | Path to CA certificate for TLS |
| `tls_client_cert` | Path to client certificate for mTLS |
| `tls_client_key` | Path to client key for mTLS |

## Platform Properties

Define execution requirements through platform properties:

```python platforms/defs.bzl
def platforms(name):
    native.platform(
        name = "linux-x86_64-large",
        constraint_values = [
            "ovr_config//os/constraints:linux",
            "ovr_config//cpu/constraints:x86_64",
        ],
        exec_properties = {
            "cpu_count": "8",
            "memory_kb": "16000000",
        },
    )
```

## Troubleshooting

### Connection errors

Verify NativeLink is running and accessible:

```bash
grpcurl -plaintext localhost:50051 list
```

### TLS handshake failures

Ensure certificates are correctly configured:

```bash
openssl s_client -connect cache.example.com:50051
```

### Platform not found

Verify platform definitions exist and are referenced correctly in `.buckconfig`.
